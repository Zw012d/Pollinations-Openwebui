# Contributing to Pollinations-OpenWebUI

Thank you for your interest in contributing to Pollinations-OpenWebUI! Whether you're looking to report issues, propose features, or improve existing code, your contributions are welcome.

## Types of Contributions

1. **Reporting Bugs**: If you encounter a bug, please create an issue in the GitHub repository with clear steps to reproduce it.
2. **Adding Features**: We welcome suggestions for new features! Please discuss your ideas in an issue before implementing them.
3. **Code Contributions**: If you wish to write code, we recommend forking the repository and creating a pull request (PR) once your changes are ready for review.

## Getting Started

1. **Fork the Repository**: Use the GitHub interface to create a personal copy of the repository.
2. **Clone Your Fork**: Clone your forked repository to your local machine.
    ```bash
    git clone https://github.com/YOUR_USERNAME/Pollinations-Openwebui.git
    cd Pollinations-Openwebui
    ```
3. **Create a New Branch**: Create a descriptive branch for your changes.
    ```bash
    git checkout -b feature/your-feature-name
    ```
4. **Make Your Changes**: Implement your feature or bug fix.
5. **Commit Your Changes**: Use clear commit messages to describe your changes.
6. **Push Your Changes**: Push your branch to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```
7. **Create a Pull Request**: Go to the original repository on GitHub and create a PR from your forked repository's branch.

## Guidelines
- Ensure your code follows the existing code style and conventions.
- Write tests to cover any new functionality.
- Keep your commits focused on a specific change and avoid large pull requests with unrelated changes.

## Code of Conduct
Please adhere to our Code of Conduct in all interactions related to this project.

Happy contributing!
